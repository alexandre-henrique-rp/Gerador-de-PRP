## Definição de Pronto (Definition of Done - DoD)
*Para uma tarefa ser considerada "Pronta", ela deve atender a TODOS os seguintes critérios:*
- [ ] Código foi revisado (Pull Request aprovado) por pelo menos um outro membro da equipe.
- [ ] Todos os testes automatizados (unitários e de integração) estão passando.
- [ ] A cobertura de testes para o novo código atinge a meta definida (>80%).
- [ ] A funcionalidade foi validada em ambiente de Staging.
- [ ] A documentação relevante (se houver) foi atualizada.
